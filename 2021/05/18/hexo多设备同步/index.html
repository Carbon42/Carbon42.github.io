hexo多设备同步

发表于 2021-05-18 分类于 hexo
配合git控制功能使用最佳

问题所在

由于有多台电脑，多设备编写hexo博客的需求就出现了。
hexo博客文件虽然会上传到GitHub仓库，但那只是hexo生成的静态网页文件，只用于网站展示。博客的源文件只留在本地。
于是如果要在多台设备上编写博客，或者担忧文件单纯保存在本地一个机器上不安全，就需要另寻出路了。
将源文件上传至git仓库是一个很好的选择，那么应该怎么做呢。

临时解决方案

一种简单的解决方法是单纯使用仓库的分支(branch)功能。

仓库的各个分支之间是相互平行的关系，也就是说彼此的文件相似，但是互不影响，直到将分支合并为止。分支功能对多人协作开发作用很大。

具体操作

博客git仓库新建分支blogsource，并设为默认分支（本地git push时不需要每一次都在指令里指明要push到的分支）。

本地博客文件夹blog其实只绑定了hexo deploy时的git仓库，所以我们将其专门用作静态博客网页文件的存放处。

我们用新建的分支来存放源文件。

将新建分支git clone url到本地（生成一个文件夹username.github.io），然后复制blog文件夹的所有文件粘贴到username.github.io文件夹。

注意，博客文件中的themes文件夹中的.git文件夹会导致上传失败，原因是要上传到仓库的文件中包含了另一个仓库的文件。也可能找不到.git文件夹，这种情况就只能对整个themes文件夹下手了。

解决这个问题可以用git的子仓库功能，但也有更粗暴的处理方法，就是.git文件夹（或themes文件夹）通过其他方式传输，git push只上传其他文件。

另一台电脑只需git clone博客仓库，并放入themes文件夹即可开始工作。（当然，这建立在这台电脑已经安装了git,node.js,hexo，并绑定了ssh key到github的前提下。

最佳解决方案：git子仓库功能

由于我们不能git push到hexo主题作者的仓库里，安装主题时不要直接 clone 作者的仓库，先 fork 主题项目，再 clone 自己的仓库里的项目。

如果发生了

三联之后网页一片空白的问题，大概率是主题文件的路径出了问题，很有可能是themes中主题文件夹名称与第一台设备上的不一致
